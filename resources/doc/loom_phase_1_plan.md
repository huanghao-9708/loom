# 织 · Loom —— 第一阶段 (MVP) 实现计划文档

> 本文档基于《织 · Loom —— BYOS 文件管理平台 · 独立开发计划书》提取并细化，作为第一阶段开发过程中的指导性任务拆解与执行标准。

## 🎯 阶段目标：文件平台内核（MVP）
**预计耗时**：约 2 个月
**核心边界**：本阶段**绝对不触碰插件系统**。一切开发资源集中于打磨“文件底座”，构建一个极快、轻量、高颜值的跨源文件浏览器与统计看板。

**可演示最终目标**：
连上坚果云与本地磁盘，在一个统一的界面下看清“有几个盘、占用多少、什么文件最多”，并能点击直接进行基础流式预览。

---

## 🛠 任务拆解与执行清单

### 1. 基础设施与 UI 底座搭建 (部分已完成)
- [x] **项目初始化**：基于 Tauri v2 + Vue 3 + TypeScript 搭建双端架构。
- [x] **无边框与窗口控制**：关闭原生边框 (`decorations: false`)，实现自定义的右上角控制点与全局拖拽域 (`data-tauri-drag-region`)。
- [x] **极简 SaaS 主题系统**：基于 Tailwind CSS v4 建立主题变量体系（纯白主调，禁用系统暗色跟随）。
- [x] **静态 UI 拼装**：完成左右分栏布局、文件浏览区、Bento（便当盒）统计面板的静态实现。

### 2. 本地数据库与状态引擎 (SQLite)
- [x] **数据库初始化**：在宿主 `AppData` 目录下建立 `loom.db`。
- [x] **表结构搭建**：
  - `sources`：连接的源元数据（状态、类型、凭证）。
  - `files`：统一文件索引缓存。
  - *已优化*：无需静态 `disk_usage` 表，利用动态 SQL 聚合实现了零延时的统计汇总。
- [x] **FTS5 全文搜索配置**：为 `files` 表建立了前缀与模糊索引，提供毫秒级搜索能力。

### 3. 统一虚拟文件系统 (Rust VFS Core)
- [x] **抽象 `Source` Trait**：定义了一套对底层透明的异步接口 (`list_dir`, `read_file`, `get_capacity` 等)。
- [x] **本地磁盘驱动**：实现上述 Trait，并接入 `sysinfo` 提供原生物理卷容量探针。
- [x] **WebDAV 客户端接入 (坚果云基准)**：
  - 基于 Rust HTTP 库接入 WebDAV 协议。
  - 攻克了极其隐蔽的 XML Namespace 前缀污染与 `Event::Empty` 自闭合标签盲区缺陷。
  - `Range` 头的流式拉取机制已完成基建。

### 4. 扫描索引引擎与并发控制
- [x] **多源挂载管理**：前端新增并完善了“添加数据源”及“彻底卸载数据源”全量交互。
- [x] **后台全量/增量扫描**：实现了一套绝对无阻塞的扫描器，使用 `tokio` 派发任务，在入库完成后通过 Tauri Emit 向前端广播变更事件。
- [x] **磁盘使用情况汇聚**：提取后端聚合的 `BentoStats` 并完成实时注入。

### 5. 核心 UI 与数据绑定 (IPC 联调)
- [x] **统一文件浏览器**：不仅完成了双向绑定，更攻克了海量数据渲染性能，实现了工业级 IntersectionObserver 虚拟长列表滚动。
- [ ] **基本文件操作**：当前已支持点击提取元数据预览，右键级（重命名、移动与复制）指令排期在后续完善。
- [x] **Bento 看板真实数据接入**：在右侧仪表盘实时渲染图表与百分比，并独创了“智能容量回退”展示机制。

### 6. 基础文件预览引擎
- [ ] **图片预览**：支持 `.jpg`, `.png`, `.svg` 等本地/云端图片的快速呈现。
- [ ] **文本与 Markdown 渲染**：将 `.md` 或 `.txt` 读取并渲染到前端专门的预览槽位。
- [ ] **PDF 基础渲染**：采用轻量的 WASM 或 PDF.js 方案实现无插件的文档查看。

## 🚧 阶段技术挑战与对策

1. **WebDAV 兼容性黑盒**：
   *应对*：MVP 阶段锁死“坚果云”作为唯一兼容与联调基准。绝不为了追求广泛兼容性而导致进度烂尾。
2. **大盘全量扫描引发的 UI 阻塞**：
   *应对*：扫描动作必须放入单独的 Rust Worker 线程执行，前端只接收进度 Event，使用 SQLite 作缓存，坚决避免直接阻塞 Tauri 的主事件循环。
3. **跨端流媒体加载速度**：
   *应对*：通过 Rust 实现本地 HTTP 服务或自定义协议响应，强制使用 HTTP Range 切片能力，实现大体积文件的“边下边看”逻辑。
